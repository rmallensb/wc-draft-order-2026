# The Bussy Inspectors — 2026 World Cup Draft Order

A static-site tracker for our 12-manager fantasy football league's World Cup draft-order game.
Each manager has three teams (one per pot); group-stage points + cumulative knockout milestones
decide who picks their fantasy football draft slot first.

Live site: `https://<your-github-username>.github.io/wc-draft-order-2026/`

## How it works

1. **Static site** in this repo's root (`index.html` + `assets/`).
2. **Match data** lives in `data/results.json`, regenerated by `sync.py` from
   [football-data.org](https://www.football-data.org/).
3. **Manual sync** — you run `python sync.py`, commit `data/results.json`, push. GitHub Pages
   reflects the new state within ~30 seconds.

No backend. No GitHub Action (yet — see "v2" below).

## One-time setup

### 1. Register a football-data.org API key

Free tier covers the 2026 WC. Sign up at <https://www.football-data.org/client/register> and copy
the API token they email you.

### 2. Configure your environment

```sh
cp .env.example .env
# edit .env, paste your key into FOOTBALL_DATA_API_KEY
```

`.env` is gitignored — your key stays local.

### 3. Install Python deps

```sh
make install
```

### 4. Fill in `data/managers.json`

Open `data/managers.json` and replace each manager's empty `pot1`/`pot2`/`pot3` strings with their
drawn teams. Team names must match the canonical `name` field in `data/teams.json` (e.g.
`"Türkiye"` not `"Turkey"`, `"USA"` not `"United States"`).

## Running a sync

```sh
make sync
```

The Makefile auto-loads `.env` so you don't need to source it. Output:
```
Fetching https://api.football-data.org/v4/competitions/WC/matches ...
  got 104 matches
Wrote /path/to/data/results.json (12 finished matches)
Top of leaderboard: Ryan @ 23 pts
```

To preview the site locally before pushing:
```sh
make serve   # then open http://127.0.0.1:8765
```

Then commit and push (you handle git):

```sh
git add data/results.json
git commit -m "sync: pull WC results"
git push
```

GitHub Pages picks up the new commit automatically.

### Optional: local cron

If you want hourly-ish syncs without thinking about it, add a crontab entry. Example for every
3 hours during the tournament:

```cron
0 */3 * * * cd ~/personal/fantasy && make sync && git add data/results.json && git commit -m "sync" --quiet && git push --quiet
```

(Drop the git lines if you'd rather review syncs manually before pushing.)

## League rules

- **12 managers**, each drawing one team from each of three pots (3 teams total per manager).
- Pots are tiered by DraftKings preseason odds:
  - **Pot 1 (Favorites)**: Spain, France, England, Brazil, Argentina, Portugal
  - **Pot 2 (Strong)**: USA, Japan, Ecuador, Mexico, Sweden, Türkiye
  - **Pot 3 (Chaos)**: Czechia, Ivory Coast, Egypt, South Korea, Australia, Iran
- Countries may repeat across managers, **but no two managers may end up with the same exact trio.**
- **Group stage scoring**: 3 for a win, 1 for a draw, +1 per goal scored, +1 for a clean sheet.
- **Knockout bonus (cumulative)**: +5 advance from group, +3 R16, +3 QF, +5 SF, +7 Final, +10 winner.
- **Total = sum across all three teams.** Highest total picks fantasy draft slot first.
- **Tie-breaker**: higher FIFA Rank Sum wins the tie (higher rank = weaker team = underdog bonus).

## Scoring rules — adjust without touching code

Edit `data/scoring.json`, re-run `sync.py`. The knockout values are CUMULATIVE — a team that reaches
the QF earns 11 total points (5 + 3 + 3), not 3.

## Troubleshooting

| Problem | Fix |
|---|---|
| `FOOTBALL_DATA_API_KEY is not set` | `source .env` first, or `export FOOTBALL_DATA_API_KEY=…` |
| `HTTP 403` from football-data.org | Free tier doesn't have this competition — check your account, the WC is on the free tier as competition `WC` |
| Warning about unmapped API team names | Open `data/teams.json` and update the `apiName` for the affected team to match the API's spelling |
| Leaderboard is empty | `data/managers.json` still has empty `pot1/pot2/pot3` strings — fill them in |
| Site loads but says "No sync has been run yet" | Run `python sync.py` |
| Want to retire a tournament after it ends | Move `data/` contents into `archive/2026/`, regenerate fresh files for the next event |
